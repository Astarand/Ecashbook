## 🔮 **Future Roadmap**

We are committed to continuously improving **E-Cashbook™**. Here are some of the key features planned for our upcoming releases:

- 🏦 **Automated Bank Reconciliation**: Upload bank statements (CSV/Excel) to automatically match and reconcile transactions with ledgers.
- 🤖 **AI-Powered Receipt OCR Scanner**: Instantly extract invoice details (vendor, date, amount, tax) from receipt photos using OCR.
- ⏰ **Smart Payment Reminders**: Automated email and SMS reminders with payment links for clients with outstanding invoices.
- 💵 **Multi-Currency Support**: Full support for international clients, enabling invoicing and transactions in multiple currencies.
- 📋 **Auto GST Return Drafts**: One-click GSTR-1 and GSTR-3B draft JSON generation based on sales and purchase history for seamless tax filing.

### 🔧 **Step-by-Step Installation**

<details>
<summary><strong>📥 1. Clone Repository</strong></summary>

```bash
git clone https://github.com/Astarand/Ecashbook.git
cd Ecashbook
```

</details>

<details>
<summary><strong>📦 2. Install Dependencies</strong></summary>

```bash
composer install
npm install
```

</details>

<details>
<summary><strong>⚙️ 3. Environment Setup</strong></summary>

```bash
cp .env.example .env
php artisan key:generate
```

</details>

<details>
<summary><strong>🗄️ 4. Database Configuration</strong></summary>

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=e_cashbook
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

</details>

<details>
<summary><strong>🏗️ 5. Run Migrations</strong></summary>

```bash
php artisan migrate
php artisan db:seed
```

</details>

<details>
<summary><strong>🎨 6. Build Assets</strong></summary>

```bash
npm run build
```

</details>

<details>
<summary><strong>🚀 7. Start Server</strong></summary>

```bash
php artisan serve
php artisan storage:link
```

</details>

## 📁 **Project Structure**

<div align="center">

### 🏗️ **Well-Organized Architecture**

</div>

```
E-Cashbook/
├── 📁 app/
│   ├── 🎮 Http/Controllers/
│   │   ├── 👑 Admin/           # Admin controllers (Subscription, CA, Customer Management)
│   │   ├── 👨‍💼 CA/              # CA controllers (Dashboard, Tasks, Compliance)
│   │   └── 👤 User/            # User controllers (Financial Management)
│   ├── 🗃️ Models/              # Eloquent models (40+ models)
│   │   ├── 👤 User.php         # User authentication & roles
│   │   ├── 🏢 Company_profiles.php  # Company information
│   │   ├── 💰 Sales.php        # Sales transactions
│   │   ├── 🛒 Purchases.php    # Purchase transactions
│   │   ├── 🏦 Assets.php       # Asset management
│   │   ├── 📉 Liabilities.php  # Liability tracking
│   │   └── ➕ ...              # Other financial models
│   ├── 🛠️ Helpers/             # Helper classes
│   │   └── 🔧 Helper.php       # Utility functions, GST API, Notifications
│   └── ⚙️ Services/            # Business logic services
├── 📁 resources/
│   ├── 🎨 views/
│   │   ├── 👑 Admin/           # Admin dashboard & management views
│   │   ├── 👨‍💼 CA/              # CA firm management views
│   │   ├── 👤 User/            # Customer financial management views
│   │   └── 🎭 App/             # Layout components (Header, Sidebar, Footer)
│   └── 🎨 assets/              # Frontend assets (CSS, JS, Images)
├── 🗄️ database/
│   ├── 📊 migrations/          # Database schema migrations
│   └── 🌱 seeders/             # Sample data seeders
├── 🌐 public/
│   └── 📦 assets/              # Compiled assets & uploads
├── 🛣️ routes/
│   └── 🌐 web.php              # Application routes (500+ routes)
└── ⚙️ config/
    ├── 🔧 app.php              # Application configuration
    ├── 🗄️ database.php         # Database configuration
    └── 🔧 custom.php           # Custom API configurations
```
